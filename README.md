# Microsoft Entra Identity And Access Operations Lab

## Overview
This project demonstrates the build, validation, and troubleshooting of a Microsoft Entra ID environment focused on identity and access management.

It covers user lifecycle management, group-based access control, administrative role assignment, and authentication behavior, with emphasis on managing cloud-only identities and enforcing access boundaries using Role-Based Access Control (RBAC) and Administrative Units.

The lab reflects real-world administrative and support scenarios, including managing user accounts, assigning permissions, validating access behavior, and troubleshooting identity-related issues such as sign-in problems and incorrect account states. It demonstrates the ability to configure identity resources, apply access controls, and make correct administrative decisions within a cloud-based environment.

All configurations and findings were validated through portal-based management, controlled testing of user actions, and verification of identity and access behavior. The project follows an evidence-first approach, with documented build validation, troubleshooting scenarios, tickets, and runbooks aligned to real IT support workflows.

## Work Performed
- Created and managed cloud-only user accounts, including provisioning, profile updates, password resets, and sign-in state control  
- Configured security groups and validated membership behavior across assigned group types  
- Applied Role-Based Access Control (RBAC) to define administrative permissions and access boundaries  
- Configured Administrative Units to evaluate scoped administration and delegated management  
- Investigated user sign-in issues by validating account status, User Principal Name (UPN), and authentication behavior  
- Verified identity and access behavior through portal-based checks and controlled testing of user actions
